## 一、核心设计思路转变

### v1.0 思路（已废弃）
> BI 看板 + 数据分析工具，OGSM 是 P3 附加功能

### v2.0 思路（当前版本）
> **软件即 OGSM 落地工具**。OGSM 一张表是产品的主界面和核心工作流，数据看板、分析模块、行动清单都是 OGSM 四个要素（O/G/S/M）的数字化支撑。

**核心逻辑**：

```
用户打开软件 → 看到的是"我的 OGSM 一张表"
                         │
              ┌──────────┼──────────┬──────────┐
              ▼          ▼          ▼          ▼
           O（目的）  G（目标）  S（策略）  M（衡量）
           战略方向   KPI进度    改善路径   行动清单
              │          │          │          │
              │     BI看板数据   四步法分析  任务跟踪
              │     自动填充     工作台      闭环机制
              │
         所有分析模块（收入/成本/顾客/多店）
         都是为了回答"G 目标是否达成"
```

## 二、OGSM 工具核心功能设计

### 2.1 OGSM 一张表（主界面）

**这是软件的核心页面，不是附加功能。**

用户进入系统后，首先看到的是本企业的 OGSM 一张表，结构如下：

```
┌─────────────────────────────────────────────────────────────┐
│  O — 目的                                                    │
│  建立数据化经营能力，实现单店盈利可复制、多店管理可扩张         │
├──────────────┬──────────────────────────────────────────────┤
│  G — 目标    │  进度                                         │
│  G1 3C体系   │  ████████░░  80%  ← 自动关联看板数据          │
│  G2 盈利改善 │  ████░░░░░░  40%  ← 净利润率实际 vs 目标      │
│  G3 成本管控 │  ██████░░░░  60%  ← 费用率实际 vs 目标        │
│  G4 顾客资产 │  ███░░░░░░░  30%  ← 复购率实际 vs 目标        │
├──────────────┼──────────────────────────────────────────────┤
│  S — 策略    │  M — 行动计划                                 │
│  S1 指标体系 │  M1.1 完成数据字典  [已完成] 财务部  4/1      │
│  S2 四步法   │  M2.1 建立对比流程  [进行中] 运营部  4/15     │
│  S3 盈利模型 │  M3.1 标杆店提取   [未开始] 财务部  5/1      │
│  S4 顾客运营 │  M4.1 会员分层设计  [进行中] 营销部  4/20     │
└──────────────┴──────────────────────────────────────────────┘
```

**关键设计原则**：
- G 目标的进度条由系统自动从 BI 数据计算，不需要手工填写
- M 行动清单的完成状态影响 S 策略的执行进度
- 点击任意 G 目标 → 跳转到对应的 BI 分析模块
- 点击任意 M 行动 → 跳转到四步法分析工作台

## 五、P1 详细需求：OGSM 一张表

### 5.1 O — 目的管理

**功能**：
- 录入企业年度/季度经营目的（定性描述，鼓舞人心）
- 支持多层级：集团目的 → 品牌目的 → 门店目的
- 目的不可量化，但必须与 G 目标关联

**字段**：
```
objective_id, tenant_id, title（目的标题）, description（详细描述）,
period（年度/季度）, level（集团/品牌/门店）, created_by, created_at
```

---

### 5.2 G — 目标管理（KPI 自动关联）

**功能**：
- 录入量化目标（KPI 指标 + 目标值 + 时间节点）
- 系统自动从 BI 数据计算当前实际值
- 自动计算完成进度（实际值 / 目标值）
- 差距预警（进度落后 > 20% 标红）

**内置 G 目标模板（来自 OGSM 框架）**：

| 模板 ID | 目标描述 | 关联 BI 指标 | 默认目标值 |
|---|---|---|---|
| G_PROFIT | 单店净利润率提升 | `net_profit_rate` | +3~5% |
| G_COST | 综合费用率下降 | `total_expense_rate` | -2~3% |
| G_MEMBER | 会员复购率提升 | `member_repurchase_rate` | +10% |
| G_EFFICIENCY | 人效提升 | `revenue_per_staff_day` | ≥1000元 |
| G_TURNOVER | 翻台率提升 | `table_turnover_rate` | +10% |
| G_SCORE | 大众点评评分 | `review_score`（手工录入） | ≥4.7 |

**字段**：
```
goal_id, objective_id, title, kpi_metric（关联 BI 指标 key）,
target_value, current_value（系统自动填充）, baseline_value,
unit, period_start, period_end, owner_role, status
```

---

### 5.3 S — 策略管理

**功能**：
- 录入实现 G 目标的关键路径（3-5 条）
- 每条策略关联一个或多个 G 目标
- 策略执行进度 = 关联 M 行动的完成率

**内置 S 策略模板**：

| 模板 ID | 策略名称 | 关联 G | 核心内容 |
|---|---|---|---|
| S_3C | 3C 指标体系建立 | G_PROFIT | 搭建自身/顾客/竞争三维指标体系 |
| S_4STEP | 四步法推行 | G_PROFIT/G_COST | 建立标准分析工作流 |
| S_MODEL | 盈利模型设计 | G_PROFIT | 三档压力测算，新店选址 |
| S_MEMBER | 顾客资产运营 | G_MEMBER | 会员分层，精准营销 |
| S_TRAIN | 团队能力培训 | G_EFFICIENCY | 店长数据分析能力培训 |

---

### 5.4 M — 行动清单（闭环跟踪核心）

**功能**：
- 行动可从四步法分析工作台直接生成（不需要手工录入）
- 每个行动关联一条 S 策略
- 行动完成后，系统自动更新关联 G 目标的进度
- 逾期行动自动标红，发送提醒

**行动状态流转**：
```
未开始 → 进行中 → 已完成
              ↓
           已逾期（截止日期过后未完成）
```

**字段**：
```
action_id, strategy_id, title（行动描述）,
what（具体做什么）, who（责任人）, when（截止日期）,
expected_result（预期效果）, actual_result（实际效果，完成后填写）,
kpi_impact（影响哪个 KPI，完成后自动触发 G 进度重算）,
status, priority, created_from（手工/四步法生成/异常预警触发）
```

## 六、P1 详细需求：BI 数据看板（为 G 服务）

### 6.1 经营总览看板

**定位变化**：不再是独立的"高管战略图"，而是 **G 目标的实时数据面板**。

每个指标卡片显示：
```
┌─────────────────────────┐
│  净利润率                │
│  实际：12.3%             │
│  目标：15.0%  ← G 目标  │
│  差距：-2.7%  ← 标红    │
│  [查看分析] [创建行动]   │  ← 直接跳转四步法 / 生成 M
└─────────────────────────┘
```

**8 个核心指标**（与 OGSM G 目标模板一一对应）：

| 指标 | 对应 G 目标 | 预警阈值 |
|---|---|---|
| 总营收 | G_PROFIT | 差异率 > 5% |
| 综合毛利率 | G_PROFIT | 差异率 > 3% |
| 食材成本率 | G_COST | 超预算标红 |
| 净利润率 | G_PROFIT | < 15% |
| 翻台率 | G_TURNOVER | 差异率 > 10% |
| 人效 | G_EFFICIENCY | < 1000元/人/天 |
| 坪效 | G_PROFIT | < 100元/㎡ |
| 会员复购率 | G_MEMBER | 差异率 > 5% |

---

### 6.2 收入分析（支撑 G2 盈利改善）

**四步法第二步"二拆数据"的数字化**：

拆解路径（操作 ≤ 3 步）：
```
总收入异动
  → 渠道拆解（堂食/外卖/自提/私域）
    → 时段拆解（早/午/下午茶/晚/夜宵）
      → 因素拆解（客单因素 + 客流因素）
```

核心计算（四步法第三步"三定归因"）：
```
收入异动 = 客单因素 + 客流因素
客单因素 = (本期客单 - 上期客单) × 上期客流
客流因素 = (本期客流 - 上期客流) × 本期客单
```

分析结果可直接生成 M 行动：
- 客流因素为主 → 建议行动：营销引流、渠道优化
- 客单因素为主 → 建议行动：菜单结构调整、套餐设计

---

### 6.3 成本费用分析（支撑 G3 成本管控）

**三线对比**（标准成本 / 理论成本 / 实际成本）：
```
原材料损益率 = (理论成本 - 实际成本) / 理论成本
```

**盈利模型阈值预警**（硬编码）：
```
利润率 < 15%    → 净利润率标红
租售比 > 15%    → 空间费用率标红
人工 > 25%      → 人员费用率标红
坪效 < 100元/㎡ → 坪效预警
人效 < 1000元/天 → 人效预警
```

## 七、P2 详细需求：四步法分析工作台

**这是 OGSM 的执行引擎**，将"发现问题 → 定位问题 → 归因 → 决策"固化为引导式工作流。

### 7.1 工作台入口

三种触发方式：
1. 从 OGSM 一张表点击偏差 G 目标 → 自动进入对应分析
2. 从 BI 看板点击异常指标 → 自动进入对应分析
3. 从异常预警通知 → 直接跳转

### 7.2 四步引导流程

**Step 1 — 一比目标**
- 自动加载当前 G 目标 vs 实际值
- 展示预算/同比/环比三维对比
- 差异率 > 5% 的指标自动标红，引导用户选择要分析的异常指标

**Step 2 — 二拆数据**
- 根据选择的指标，推荐拆解维度（渠道/时段/商圈/顾客类型）
- 逐层下钻，每层展示贡献度排名
- 自动标注"主要缺口来源"

**Step 3 — 三定归因**
- 自动计算客单因素 / 客流因素
- 提供归因假设模板（偶发性 vs 系统性）
- 用户确认主要归因后进入下一步

**Step 4 — 四做决策**
- 根据归因结果，推荐行动模板（来自历史成功案例库）
- 用户选择/编辑行动 → 填写 Who/When/预期效果
- 一键生成 M 行动清单，自动关联到 OGSM 一张表

## 八、P2 详细需求：顾客资产分析（支撑 G4）

### 8.1 核心指标

| 指标 | 计算方式 | 对应 G 目标 |
|---|---|---|
| 会员复购率 | 30/60/90 天内复购客户数 / 总客户数 | G_MEMBER |
| 新客占比 | 首次消费客户数 / 总客户数 | G_MEMBER |
| 会员贡献度 | 会员收入 / 总收入 | G_MEMBER |
| 沉睡会员数 | 90 天未消费的会员数 | G_MEMBER |
| 客诉率 | 投诉数 / 总订单数 | G_SCORE |

### 8.2 分析输出

- 新老客结构变化趋势（月度）
- 会员复购率漏斗（首次→二次→三次→稳定复购）
- 沉睡会员名单导出（用于 CRM 触达）
- 分析结果可直接生成 M 行动（如：制定沉睡会员唤醒方案）

## 九、P3 详细需求：盈利模型沙盘（支撑 G6）

### 9.1 核心公式（来自方法论）

```
理想面积 = 日客流量 / 上座次数 × 单个餐位面积
实际座位数 = 理想面积 / 单个餐位面积
用工人数 = 店面积 / 人均服务面积

三档压力测算：
  乐观档（最好月份客流）→ 净利润率 / 投资回收周期
  正常档（历史平均客流）→ 净利润率 / 投资回收周期
  保守档（最差月份客流）→ 净利润率 / 投资回收周期
```

### 9.2 功能设计

- 从标杆门店自动提取常量参数（客单价/毛利率/人均服务面积）
- 用户输入变量（预计客流/面积/租金）
- 实时计算三档结果，对不达标指标标红
- 支持多方案对比（同一选址不同面积/租金方案）
- 结果可生成 M 行动（如：调整选址方案）

## 十一、非功能性需求

### 11.1 性能

| 场景 | 要求 |
|---|---|
| OGSM 一张表加载 | P95 < 1s（G 进度实时计算） |
| BI 看板首屏 | LCP < 2.5s |
| 指标查询（单店，30天） | P95 < 200ms |
| 多店对比（20家，90天） | P95 < 3s |
| 四步法分析（含因素拆解） | P95 < 500ms |
| 数据导入（5MB Excel） | P95 < 30s（含质量检查） |

### 11.2 可用性

- OGSM 一张表：核心操作（录入 G 目标 / 创建 M 行动）≤ 3 步
- 四步法工作台：4 个步骤，每步有明确引导，不需要培训即可使用
- 所有分析结果提供"一键生成行动"按钮，降低决策门槛

### 11.3 数据安全

- OGSM 数据（目标/策略/行动）按角色权限控制
- G 目标数据仅 owner/manager 可见，staff 只看自己的 M 行动
- 导出报告包含水印（租户名 + 导出时间）
