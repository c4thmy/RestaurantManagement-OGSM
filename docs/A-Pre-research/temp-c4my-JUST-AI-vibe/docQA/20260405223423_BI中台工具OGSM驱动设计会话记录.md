### Q4：重点围绕 OGSM 工具思路设计软件，再追加完善更新已生成相关全部文件

**核心设计转变**：
- v1.0 思路：BI 看板 + 数据分析工具，OGSM 是 P3 附加功能
- v2.0 思路：**软件即 OGSM 落地工具**，OGSM 一张表是主界面，BI 数据看板为 G 目标进度提供自动计算支撑，四步法工作台是 OGSM 的执行引擎

**OGSM 数据流向（不可打破）**：
```
BI 数据导入 → G 目标进度自动更新 → 偏差触发四步法分析
     ↑                                        ↓
M 行动完成 ←── 行动清单闭环跟踪 ←── Step 4 生成 M 行动
```

## 关键设计决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| OGSM 一张表定位 | 主界面（首页路由） | 软件核心价值是 OGSM 落地，不是 BI 看板 |
| G 目标进度计算 | 系统自动从 BI 数据计算 | 手工填写负担重且不准确，失去自动化价值 |
| 四步法工作台触发方式 | 3 种（OGSM 偏差/BI 异常/手动） | 降低使用门槛，让分析自然发生 |
| 行动推荐 v1 | 规则引擎（非 AI） | 快速交付，可解释，不依赖外部 API |
| 多租户隔离 | PostgreSQL 独立 Schema | 隔离强度与运维成本平衡 |
| 分析会话持久化 | 数据库存储草稿 | 支持中途退出继续，支持团队复盘 |
